# DESIGN_GUIDELINES.md

DeckDeckDeck Android Native MVP의 화면, 색상, 카드, 상호작용 디자인 기준을 정의한다.

제품 요구사항은 `docs/PRD.md`를 기준으로 하고, 디자인 관련 판단은 이 문서를 우선 확인한다.

---

## 0. 용어 정의

이 문서에서 `카드`라는 말은 기본적으로 사용자가 저장한 실행 항목인 `액션 카드`를 뜻한다.
카드 모양을 가진 다른 UI와 헷갈릴 수 있으므로, 아래 용어를 구분해서 쓴다.

| 용어 | 의미 | 탭 동작 |
|---|---|---|
| 액션 카드 | 문구 복사, 웹사이트 열기처럼 저장된 실행 항목 | 저장된 액션 실행 |
| 카테고리 카드 | 액션 카드를 묶는 카테고리를 카드 모양으로 보여주는 항목 | 해당 카테고리의 액션 카드 목록으로 이동 |
| 빈 슬롯 | 새 액션 카드나 카테고리를 추가하기 위해 비워 둔 자리 | 생성 흐름 시작 |
| 카드형 항목 | 액션 카드, 카테고리 카드, 빈 슬롯을 함께 부를 때 쓰는 디자인 용어 | 항목 역할에 따라 다름 |
| 카드 Surface | 둥근 사각형, 약한 elevation, padding처럼 카드형 항목에 쓰는 시각 스타일 | 자체 동작 없음 |

일반 컨테이너는 `카드`라고 부르지 않는다.
오버레이의 바깥 틀은 `오버레이 패널`, 입력 UI는 `입력 영역`, 하단 임시 화면은 `시트`라고 부른다.

## 1. 디자인 방향

### 1.1 시각 방향

원본 DeckDeckDeck의 감각을 유지하되, 모바일에서 조작하기 쉽게 바꾼다.

핵심 키워드:

```text
Soft Sky Pastel
Soft Keycap Card
Local Personal Launcher
Quick Tap Action
```

### 1.2 기본 팔레트

메인 테마 색은 `#a1c6ea`를 기준으로 한다.

| 역할             | 색상        | 용도                     |
| -------------- | --------- | ---------------------- |
| Main           | `#a1c6ea` | 주요 브랜드 색, 선택 상태, 주요 버튼 |
| Main Accent    | `#2f76b7` | Main보다 짙은 활성 상태 보조 강조, 연결 상태 칩 |
| Background     | `#f6faff` | 앱 전체 배경                |
| Surface        | `#ffffff` | 카드형 항목, 입력 영역, 시트      |
| Surface Subtle | `#eaf4ff` | 빈 슬롯, 보조 영역, 선택 전 상태   |
| Text Primary   | `#1f2a37` | 주요 텍스트                 |
| Text Secondary | `#5f6f82` | 설명, 라벨, 보조 텍스트         |
| Accent         | `#f7c873` | 강조 액션, 작은 포인트          |
| Success        | `#72c7a5` | 성공 상태                  |
| Error          | `#f28b82` | 오류 상태                  |
| Border         | `#c8dced` | 카드형 항목 경계, 구분선         |

### 1.3 색상 사용 원칙

1. 화면 전체는 `Background`를 기본으로 사용한다.
2. 카드형 항목과 입력 영역은 `Surface`를 사용해 내용이 먼저 보이게 한다.
3. 주요 실행 버튼, 선택된 탭, 활성 상태는 `Main`을 사용한다.
4. `Accent`는 화면의 한두 지점에서만 보조 강조로 사용한다.
5. 오류와 성공은 각각 `Error`, `Success`로 구분한다.
6. 텍스트는 `Text Primary`와 `Text Secondary`를 우선 사용하고, 색상만으로 의미를 전달하지 않는다.

### 1.4 카드형 항목 형태

| 요소 | 방향 |
|---|---|
| 모서리 | 둥근 사각형 |
| 그림자 | 약한 elevation |
| 눌림감 | 탭 시 살짝 눌리는 느낌 |
| 여백 | 모바일 터치에 충분한 padding |
| 텍스트 | 짧은 제목 중심 |
| 이미지 | 선택 이미지 또는 타입 아이콘 |

### 1.5 오버레이 화면 기준

오버레이는 일반 앱 화면이 아니라, 현재 화면 위에 잠깐 떠서 빠르게 실행하는 작은 런처 패널이다.

설정 화면처럼 전체 화면을 꽉 채우는 관리 UI보다, 아래 기준을 우선한다.

1. 패널은 화면 단계에 맞는 배경색을 약간 투명하게 사용해 뒤 화면과 분리하되 너무 무겁게 보이지 않게 한다.
2. 색상은 넓은 면보다 액션 카드 라벨, 선택 상태, 주요 액션에 작게 사용한다.
3. 카드형 항목은 한눈에 훑고 바로 누를 수 있도록 조밀하게 배치한다.
4. 오버레이 안에서는 설명 문구보다 짧은 제목과 라벨을 우선한다.
5. 설정, 닫기, 뒤로가기 같은 보조 액션은 아이콘 중심으로 둔다.

카테고리 카드 선택 후 액션 카드 목록 화면에 들어가면, 패널 배경을 카테고리 카드와 같은 하늘색 계열로 바꾼다.
이는 사용자가 단순히 목록이 바뀐 것이 아니라, 선택한 카테고리 안쪽으로 들어와 있다는 느낌을 받게 하기 위한 디자인 신호이다.

오버레이 팔레트 사용 예시는 다음을 기준으로 한다.

| UI 요소                  | 권장 표현                                               |
| ---------------------- | --------------------------------------------------- |
| 카테고리 목록 화면의 오버레이 패널 배경 | `Surface #ffffff`, alpha 약 `0.96`                   |
| 액션 카드 목록 화면의 오버레이 패널 배경 | `Background #f6faff`, alpha 약 `0.96`                |
| 오버레이 패널 형태             | 8dp 둥근 사각형, 약한 elevation                            |
| 오버레이 패널 테두리            | 카테고리 목록 화면: `Main #a1c6ea`, alpha 약 `0.30`<br>액션 카드 목록 화면: `Main #a1c6ea`, alpha 약 `0.65` |
| 카테고리 목록 화면의 카테고리 카드 배경 | `Surface #ffffff`                                   |
| 액션 카드 목록 화면의 액션 카드 배경  | `Surface Subtle #eaf4ff`                            |
| 액션 카드 목록 화면의 빈 슬롯 배경   | `Surface Subtle #eaf4ff`                            |
| 카드형 항목 테두리             | `Border #c8dced`                                    |
| 카드형 항목 제목              | `Text Primary #1f2a37`                              |
| 액션 카드 라벨               | `Main #a1c6ea`                                      |
| 카테고리 라벨                | `Text Secondary #5f6f82`                            |
| 비활성 액션 카드              | opacity 약 `50%`, 라벨은 `Text Secondary #5f6f82`       |
| 보조 아이콘                 | `Text Primary #1f2a37` 계열                           |

오버레이의 액션 카드 목록 화면에서는 색상 역할을 다음처럼 구분한다.

1. 앱 전체 기본 배경은 `Background #f6faff`이다.
2. 액션 카드 목록 화면의 오버레이 패널 배경은 `Background #f6faff`에 alpha 약 `0.96`을 적용한다.
3. 액션 카드와 `+ 카드 추가` 빈 슬롯의 내부 배경은 `Surface Subtle #eaf4ff`를 사용한다.
4. `Surface Subtle #eaf4ff`는 `Background #f6faff`와 다른 색이며, 눈으로 볼 때 더 푸른 연한 하늘색이다.

## 2. 카드형 항목 상태

| 상태 | 설명 |
|---|---|
| Default | 일반 액션 카드 또는 카테고리 카드 |
| Pressed | 탭 중인 카드형 항목 |
| Empty | 추가 가능한 빈 슬롯 |
| Disabled | 비활성 액션 카드 |
| Error | 실행 실패 상태 메시지 |
| Success | 실행 성공 상태 메시지 |

### 2.1 Empty 상태

빈 슬롯은 다음처럼 표시한다.

```text
+ 카드 추가
```

또는

```text
+ 카테고리 추가
```

### 2.2 Disabled 상태

비활성 액션 카드는 삭제된 것이 아니라 잠시 사용할 수 없는 상태로 보여야 한다.

권장 표현:

1. opacity 낮춤
2. 실행 불가 아이콘 또는 작은 라벨
3. 탭 시 `비활성화된 카드입니다` 메시지 표시

오버레이에서는 비활성 액션 카드도 같은 위치에 남겨 사용자가 카드가 사라졌다고 느끼지 않게 한다.

액션 카드 전체는 약하게 보이게 하고, 라벨은 `비활성 · 문구`처럼 상태와 타입을 함께 보여준다.

## 3. Android UX 고려사항

### 3.1 한 손 조작

1. 주요 액션은 화면 하단 가까이에 배치한다.
2. Floating Action Button 사용을 고려한다.
3. 카드형 항목 크기는 최소 64dp 이상으로 한다.
4. 긴 텍스트 편집 화면은 스크롤 가능해야 한다.

오버레이는 한 손 조작을 위해 사용자의 손 방향에 따라 좌우에 붙을 수 있다.

오버레이 폭은 전체 화면을 다 쓰기보다 화면 일부만 차지해, 뒤 화면의 맥락을 가리지 않게 한다.

### 3.2 탭과 편집 분리

액션 카드 탭은 실행이다.

카테고리 카드 탭은 해당 카테고리의 액션 카드 목록으로 이동한다.

편집은 길게 누르기 또는 메뉴 버튼으로 분리한다.

| 행동 | 의미 |
|---|---|
| 짧게 탭 | 액션 카드 실행 또는 카테고리 이동 |
| 길게 누르기 | 편집 메뉴 |
| `⋮` 메뉴 | 편집/복사/이동/삭제 |

### 3.3 실행 후 피드백

| 실행 | 피드백 |
|---|---|
| 문구 복사 성공 | `복사했습니다` |
| 문구 없음 | `복사할 문구가 없습니다` |
| 웹사이트 열기 성공 | 별도 메시지 생략 가능 |
| 웹사이트 열기 실패 | `웹페이지를 열지 못했습니다` |
