# PRD: DeckDeckDeck Android Native MVP

- 문서 버전: v0.2
- 작성일: 2026-06-27
- 대상 플랫폼: Android
- 구현 방식: Kotlin Native Android 앱
- MVP 범위: 오버레이 실행, 문구 저장/복사, 웹사이트 저장/열기

---

## 1. 제품 정의

DeckDeckDeck Android는 자주 쓰는 문구와 자주 여는 웹사이트를 카드로 저장하고, 앱 실행 시 뜨는 오버레이에서 탭 한 번으로 복사하거나 여는 로컬 개인 런처 앱이다.

핵심 목표는 다음 세 가지이다.

1. 반복해서 쓰는 문구를 빠르게 클립보드에 복사한다.
2. 자주 여는 웹사이트를 빠르게 브라우저로 연다.
3. 서버, 계정, 클라우드 없이 기기 안에서 카테고리와 카드를 관리한다.

---

## 2. 핵심 범위

MVP에서 지원하는 실행 타입은 두 가지뿐이다.

1. 문구 복사
2. 웹 주소 열기

문구 카드는 다른 앱의 현재 입력 위치에 직접 붙여넣지 않는다. Android 일반 앱 권한으로는 다른 앱 입력창에 임의 입력할 수 없으므로, MVP에서는 클립보드 복사만 지원한다.

웹 주소는 저장 시 정규화한다.

| 입력 | 저장/처리 |
|---|---|
| `google.com` | `https://google.com` |
| `https://google.com` | 그대로 저장 |
| `http://localhost:3000` | 허용 |
| `192.168.0.1` | `https://192.168.0.1` |
| `hello world.com` | 실패 |
| `ftp://example.com` | 실패 |
| `example` | 실패 |

## 3. 구현 완료

현재 코드 기준으로 완료된 항목이다.

- Room 기반 로컬 저장소를 사용한다.
- 카테고리를 생성, 조회, 수정, 삭제할 수 있다.
- 문구 카드와 웹사이트 카드를 생성, 조회, 수정, 삭제할 수 있다.
- 문구 카드 실행 시 저장된 문구를 클립보드에 복사한다.
- 웹사이트 카드 실행 시 저장된 URL을 브라우저로 연다.
- URL 저장 시 `https://` 자동 보정과 http/https 검증을 수행한다.
- 제목 없음, 문구 없음, URL 없음, URL 형식 오류를 저장 단계에서 막는다.
- 비활성 카드 실행을 막고 안내 메시지를 표시한다.
- 카드 활성/비활성 상태를 변경할 수 있다.
- 앱 실행 시 일반 관리 화면 대신 오버레이 실행 흐름으로 진입한다.
- 오버레이에서 카테고리 목록과 선택한 카테고리의 카드 목록을 볼 수 있다.
- 오버레이에서 문구 복사 또는 웹사이트 열기 실행 후 오버레이 서비스를 정리한다.
- 오버레이의 설정 아이콘으로 관리 화면에 진입할 수 있다.
- 관리 화면 뒤로가기 시 오버레이로 복귀한다.
- 오버레이 권한이 없으면 권한 안내 화면을 보여주고 Android 설정 화면으로 이동할 수 있다.
- 오버레이 표시 실패 시 앱이 크래시 나지 않고 안내 후 서비스를 정리한다.

검증된 자동 확인:

```powershell
.\gradlew.bat :app:testDebugUnitTest
.\gradlew.bat :app:assembleDebug
```

## 4. 부분 구현

기본 동작은 있으나 PRD의 최종 기대치와 차이가 있는 항목이다.

- 카테고리 수정은 이름 중심으로 동작한다. 설명, 이미지, 활성/비활성 편집 UI는 아직 부족하다.
- 카드 수정은 제목, 타입, 내용, URL, 활성/비활성을 다룬다. 설명과 이미지는 아직 다루지 않는다.
- 카드 생성은 별도 “빈 카드 탭 → 생성 화면” 흐름이 아니라 관리 화면 안의 인라인 입력 폼으로 제공된다.
- 카드 편집은 별도 전체 화면이 아니라 메뉴와 다이얼로그 중심으로 제공된다.
- 오버레이와 관리 화면의 카드는 기본 카드 UI로 표시된다. 사용자 지정 이미지나 썸네일은 아직 없다.
- 오버레이 뒤에 직전 화면이 자연스럽게 보이는지는 실제 기기에서 추가 확인이 필요하다.
- 권한 안내 흐름은 구현되어 있으나 제조사별 Android 설정 화면에서의 복귀 동작은 실제 기기 검증이 필요하다.

## 5. 미구현

MVP 기준으로 아직 구현되지 않은 항목이다.

- 카테고리 설명 입력/수정
- 카테고리 이미지 설정
- 카테고리 활성/비활성 UI
- 카드 설명 입력/수정
- 카드 이미지 설정
- 빈 카드 탭을 통한 카드 생성 UX
- 긴 누르기 기반 카드 편집 UX
- 설정 화면의 앱 정보
- 설정 화면의 로컬 저장 안내
- 설정 화면의 오버레이 권한 안내 섹션
- 설정 화면의 문구 복사 동작 안내
- 데이터 전체 삭제
- 실제 기기 수동 검증 체크리스트 완료

실제 기기에서 확인해야 할 항목:

1. 앱 아이콘 실행 시 일반 관리 화면이 먼저 보이지 않는지 확인한다.
2. 오버레이 권한이 꺼져 있을 때 안내 화면이 뜨는지 확인한다.
3. 권한을 켠 뒤 돌아오면 오버레이가 바로 뜨는지 확인한다.
4. 문구 카드 실행 후 클립보드에 복사되는지 확인한다.
5. 웹사이트 카드 실행 후 브라우저가 열리는지 확인한다.
6. 카드 실행 후 원래 화면으로 돌아가는지 확인한다.
7. 오버레이 설정 아이콘으로 관리 화면에 진입하고 뒤로가기 시 오버레이로 복귀하는지 확인한다.

## 6. MVP 제외 범위

다음 기능은 현재 MVP에 넣지 않는다.

- 핫키
- 작은 플로팅 핸들
- 현재 커서 직접 붙여넣기
- IME 구현
- 접근성 서비스 구현
- 파일/폴더 실행
- 앱 실행 카드
- 미디어 제어
- 터미널 명령
- Galaxy Buds 연결/해제 자동화
- 백업/복원
- 클라우드 동기화
- 로그인/계정 시스템

---

## 7. 다음 구현 후보

우선순위가 높은 다음 작업은 다음과 같다.

1. 실제 기기에서 오버레이 권한, 실행, 종료 흐름을 수동 검증한다.
2. 설정 화면에 앱 정보, 로컬 저장 안내, 문구 복사 안내를 추가한다.
3. 카테고리/카드 설명 필드를 UI에 노출한다.
4. 빈 카드 탭 기반 생성 흐름으로 편집 UX를 정리한다.
5. 이미지 선택/썸네일 기능을 검토한다.
